# ToBid 激励视频 H5 测试页使用说明

## 你拿到的文件

核心文件是：

```text
index.html
```

这是一个纯 H5 页面，不需要 npm，不需要打包，不需要安装依赖。

## 放到 App 里怎么测

把 `index.html` 上传到一个 App 可以访问的 HTTPS 地址，然后让 Android App 用内置 WebView 打开这个地址。

不要用手机浏览器直接打开，因为手机浏览器没有 App 的原生广告桥，无法展示 ToBid 激励视频。

## 页面里的按钮怎么点

推荐顺序：

1. 点“检测桥接”
   - 成功：页面显示“已检测到 WebViewJavascriptBridge”
   - 失败：说明这个新页面没有拿到 App 注入的 JSBridge，需要客户端确认是否允许这个页面使用桥

2. 点“自动获取并预加载”
   - 页面会调用旧 H5 同款方法：

```js
adRequest
preLoadVideo
```

   - 如果接口返回了激励视频配置，页面会自动填入 `sdk_id`

3. 点“展示激励视频”
   - 页面会调用：

```js
connectVideo
```

   - 传入结构和旧 H5 保持一致：

```js
{
  id: "leagueCollect_14",
  data: {},
  origin: "...",
  sdk_id: "...",
  imp_mon_arr: [],
  click_mon_arr: [],
  ct: "",
  pgid: "1.15.2"
}
```

## 如果你已经查到了 sdk_id

可以不走自动获取。

直接在页面里填写：

```text
sdk_id
origin
```

然后点：

```text
手动 sdk_id 预加载
展示激励视频
```

其中 `sdk_id` 大概率就是你要核对的 ToBid placementId。

## URL 参数写法

也可以把参数直接拼在页面 URL 后面。

示例：

```text
https://你的域名/tobid-h5-reward-demo/index.html?sdk_id=你的sdk_id&origin=广告源&pgid=1.15.2
```

如果想打开页面后自动预加载：

```text
https://你的域名/tobid-h5-reward-demo/index.html?sdk_id=你的sdk_id&origin=广告源&auto_load=1
```

如果想自动预加载后再自动展示：

```text
https://你的域名/tobid-h5-reward-demo/index.html?sdk_id=你的sdk_id&origin=广告源&auto_load=1&auto_show=1
```

## 失败时怎么看

页面底部有“运行日志”。

常见结果：

```text
未检测到 WebViewJavascriptBridge
```

说明新页面没有拿到 App 的 JSBridge。这个不是 H5 能单独解决的，需要客户端确认 WebView 是否注入桥。

```text
adRequest 没找到 ad_type=sdk_rewarded_video 的广告
```

说明广告配置接口没有返回激励视频。可以抓包看 `/plat/v3?pgid=1.15.2` 或 `/plat/v3?pgid=8.15.2` 的返回。

```text
preLoadVideo 回调超时
```

不一定失败。有些原生方法可能没有回调。继续点“展示激励视频”看是否能拉起广告。

```text
connectVideo 回调超时
```

也不一定失败。如果广告已经弹出，说明调用成功但没有 JS 回调。

## 关键判断

这个 H5 页面能否展示激励视频，取决于 Android App 是否已经支持下面三个桥接方法：

```text
adRequest
preLoadVideo
connectVideo
```

如果 App 支持，H5 可以触发激励视频。

如果 App 不支持，仅靠 H5 不能直接请求 ToBid SDK。
